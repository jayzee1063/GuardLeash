# GuardLeash

Специально для сервера Lunify

GuardLeash — Paper-плагин для полицейской механики связывания игроков обычным ванильным поводком. Плагин не ломает стандартную механику поводков для животных: игроки могут связывать других игроков только при наличии специального права.

## Как работает механика

- Игрок с правом `guardleash.use` может кликнуть обычным поводком по другому игроку и привязать его к себе.
- У связанного игрока блокируются:
  - самостоятельное движение;
  - прыжки;
  - атаки;
  - взаимодействия ПКМ/ЛКМ;
  - ломание и установка блоков;
  - выбрасывание предметов;
  - взаимодействие с инвентарём;
  - внешние телепорты.
- Ведущий игрок может тянуть связанного за собой. Подтягивание сделано через мягкую velocity-механику, поэтому игрок не должен ощущать постоянные телепорты.
- Если связанного тянут на блок, плагин сам даёт ему служебный подъёмный импульс, чтобы игрок не застревал на ступеньках.
- Связанного игрока можно привязать к забору кликом по забору.
- При развязывании поводок не выпадает на землю, а возвращается в инвентарь.
- Если инвентарь заполнен, возврат поводка ставится в очередь и будет выполнен позже.
- Если связанный игрок вышел и зашёл обратно, плагин восстановит связь, если ведущий находится рядом в настроенном радиусе.
- Если восстановить связь нельзя, поводок возвращается владельцу.

## Права

| Право | Назначение |
| --- | --- |
| `guardleash.use` | Позволяет связывать и развязывать игроков обычным поводком. |
| `guardleash.admin` | Позволяет перезагружать конфиг и принудительно развязывать игроков. |
| `guardleash.bypass` | Игрока с этим правом нельзя связать. |

## Команды

| Команда | Право | Описание |
| --- | --- | --- |
| `/guardleash reload` | `guardleash.admin` | Перезагружает конфиг плагина. |
| `/guardleash release <player>` | `guardleash.admin` | Принудительно снимает связь с игрока. |

Алиасы команды:

- `/gleash`
- `/lunifyguardleash`

## Конфиг

Конфиг находится в `plugins/GuardLeash/config.yml`.

### `settings`

Основные настройки:

- `use-permission` — право на использование механики.
- `bypass-permission` — право иммунитета от связывания.
- `leash-item` — предмет для связывания. По умолчанию `LEAD`.
- `consume-lead` — забирать ли поводок при связывании.
- `consume-lead-in-creative` — забирать ли поводок в креативе.
- `allow-offhand` — разрешить использование поводка из второй руки.
- `only-owner-can-release` — разрешить развязывание только владельцу связи или администратору.
- `return-lead-to-releaser` — возвращать поводок тому, кто развязал, а не тому, кто связал.
- `update-interval-ticks` — частота обновления сессий поводка.
- `proxy-y-offset` — вертикальное смещение невидимой прокси-сущности, к которой визуально крепится поводок.

### `holder-leash`

Настройки поведения, когда игрок привязан к другому игроку:

- `follow-distance` — дистанция, после которой связанный начинает подтягиваться к ведущему.
- `max-step-per-update` — максимальная скорость подтягивания за обновление.
- `pull-base-speed` — базовая скорость мягкой тяги.
- `pull-speed-per-block` — усиление тяги за каждый лишний блок дистанции.
- `step-up-velocity` — вертикальный импульс для подъёма на блоки.
- `step-check-distance` — расстояние проверки блока перед связанным.
- `movement-away-tolerance` — допуск на попытку отхода от ведущего.
- `dampen-near-holder` — гасить остаточную скорость, когда связанный уже рядом.
- `release-if-too-far` — разрывать ли связь при слишком большой дистанции.
- `release-distance` — дистанция разрыва, если включён `release-if-too-far`.

### `fence-leash`

Настройки привязки к забору:

- `enabled` — включает или выключает привязку к забору.
- `bind-all-held-targets` — если ведущий держит нескольких игроков, привязывать ли всех к забору одним кликом.
- `extra-fence-materials` — дополнительные материалы, которые считаются забором.

### `restrictions`

Ограничения для связанного игрока:

- `prevent-movement` — запрещает самостоятельное движение.
- `allow-looking-around` — разрешает вращать камерой.
- `disable-target-walk-speed` — временно обнуляет walk/fly speed связанного игрока.
- `prevent-jump` — запрещает самостоятельный прыжок.
- `prevent-inventory-interact` — запрещает взаимодействие с инвентарём.
- `prevent-damage` — запрещает наносить урон.
- `prevent-interact` — запрещает ПКМ/ЛКМ взаимодействия.
- `prevent-block-break` — запрещает ломать блоки.
- `prevent-block-place` — запрещает ставить блоки.
- `prevent-item-drop` — запрещает выбрасывать предметы.
- `prevent-teleport` — запрещает внешние телепорты.

### `safety`

Защитные настройки:

- `release-when-target-quits` — снимать или переводить в ожидание связь при выходе связанного игрока.
- `release-when-holder-quits` — снимать связи, если вышел ведущий игрок.
- `restore-target-leash-on-reconnect` — восстанавливать связь при повторном входе связанного игрока.
- `reconnect-restore-radius` — радиус восстановления связи после релога.
- `reconnect-timeout-seconds` — сколько секунд хранить ожидающую связь.
- `reconnect-restore-delay-ticks` — задержка после входа перед проверкой восстановления.
- `return-lead-on-auto-release` — возвращать поводок при автоматическом снятии связи.
- `keep-pending-leads-when-inventory-full` — не дропать поводок, если инвентарь заполнен, а поставить возврат в очередь.

### `proxy-entity`

Внутренняя невидимая сущность, к которой крепится визуальный поводок:

- `type` — тип прокси-сущности. По умолчанию `ZOMBIE`.
- `use-baby-zombie` — делать прокси-зомби маленьким, чтобы поводок крепился ниже головы игрока.

### `messages`

Все сообщения и actionbar-уведомления настраиваются через MiniMessage.

## Сборка

Плагин рассчитан на Java 21 и Paper API.

```bash
mvn clean package
```

Готовый jar будет находиться в `target/GuardLeash-1.0.0.jar`.
